# אבטחת מידע ב‑LS Duet
**מסמך טכני ללקוחות ארגוניים**
*עדכון אחרון: אפריל 2026*

## 1. תקציר מנהלים
LS Duet בנוי בגישת **Local-First**: רוב המידע שלך נשמר על המכשיר שלך עצמו, לא בענן שלנו. בענן נשמר רק המינימום התפעולי ההכרחי, באזור **האיחוד האירופי (Frankfurt)**, בתשתית **Google Cloud / Firebase**. כל המידע מוצפן במעבר (TLS) ובאחסון (AES‑256), הגישה אליו מוגבלת אך ורק לבעלי החשבון, ותוכן ההודעות נמחק מהשרת **אוטומטית תוך 48 שעות**. לא נעשה שימוש בתוכן שלך לפרסום, פרופיילינג, או אימון מודלים, ולא נמכר לאף צד שלישי.

## 2. ארכיטקטורה במבט-על

| שכבה | טכנולוגיה | היכן רץ |
|---|---|---|
| אפליקציית מובייל | React Native + Expo (iOS/Android) | המכשיר של המשתמש |
| בסיס נתונים מקומי | SQLite (מוצפן ע"י ה‑OS) | כחלק מהאפליקציה במכשיר |
| Backend | Firebase Cloud Functions (Node.js 22) | Google Cloud — `europe-west3` (Frankfurt) |
| בסיס נתונים בענן | Cloud Firestore | Google Cloud — `europe-west3` (Frankfurt) |
| גשר WhatsApp | WAHA | שרתים מנוהלים שלנו |
| תרגום | Anthropic Claude API | תקשורת ישירה Google Cloud ↔ Anthropic |
| אימות | Firebase Authentication | Google Identity |
| משלוח מתוזמן | Google Cloud Tasks | Google Cloud — `europe-west3` |

**אין לנו שרתים פיזיים, אין SSH, אין מכונות וירטואליות לתחזוקה ידנית** — כל ה‑Backend הוא Serverless ומנוהל ע"י Google Cloud.

## 3. תקשורת מאובטחת

- **כל ערוצי התקשורת**: HTTPS / TLS 1.2+ בלבד. אין נתיב לא־מוצפן.
- **כל קריאה ל‑Cloud Function** דורשת **Firebase ID Token תקף** (JWT חתום) בכותרת `Authorization: Bearer …`. הטוקן מאומת בכל קריאה דרך `admin.auth().verifyIdToken()` ב‑Google.
- **Webhook נכנס מ‑WAHA**: מאומת בסוד משותף (`x-waha-webhook-secret`) — בקשות ללא הסוד נדחות עם 401.
- **קריאות יוצאות ל‑WAHA**: עוברות מהענן שלנו בלבד, עם `X-Api-Key` פר‑סשן. מפתחות ה‑API לעולם לא נחשפים ללקוח.
- **Cloud Tasks → Cloud Functions** (להפעלת הודעות מתוזמנות): מאומת באמצעות **OIDC token** של חשבון שירות ייעודי.
- **דשבורד אדמין**: Google Sign‑In + רשימת מאושרים סגורה (email whitelist) הנאכפת בצד השרת.

## 4. אימות והרשאות

### 4.1 אימות משתמש
- Firebase Authentication עם בחירה בין: **Google Sign‑In**, **Sign in with Apple** (iOS), אימייל/סיסמה.
- תקני אבטחה של Google: זיהוי דו־שלבי תלוי בהגדרות חשבון Google של המשתמש.

### 4.2 הפרדת משתמשים בענן (Firestore Security Rules)
ברירת המחדל היא **deny‑by‑default**. אכיפה כפולה — גם בכללי האבטחה וגם ב‑Cloud Function:

| אוסף | קריאה | כתיבה |
|---|---|---|
| `users/{uid}/...` | רק בעל ה‑uid | רק בעל ה‑uid |
| `whatsappMessages` | **חסום ללקוח** — רק דרך Cloud Function שמסננת לפי uid | חסום ללקוח |
| `translationSettings` | חסום ללקוח | חסום ללקוח |
| `scheduledMessages` | חסום ללקוח | חסום ללקוח |
| `wahaAssignments/{uid}` | רק בעל ה‑uid | חסום ללקוח (רק שרת) |
| `pushTokens/{uid}` | רק בעל ה‑uid | חסום ללקוח |
| `licenses/{uid}` | רק בעל ה‑uid | חסום ללקוח |
| כללי | חסום | חסום |

המשמעות: גם אם תוקף ישיג טוקן מזויף, אין נתיב פיזי לקרוא או לכתוב מידע של משתמש אחר.

## 5. כיצד השרתים מאובטחים

הענן שלנו רץ במלואו על **Google Cloud Platform**. אנחנו נהנים אוטומטית מהבקרות והתעודות של Google:

- **תקני תאימות**: ISO 27001, ISO 27017, ISO 27018, SOC 1/2/3, PCI‑DSS, GDPR, HIPAA‑ready.
- **הצפנת דיסק במנוחה**: AES‑256 על כל הדיסקים והגיבויים, אוטומטית ושקופה.
- **הקשחת רשת**: ה‑Cloud Functions רצים בסביבת ריצה מבודדת; אין יציאות פתוחות מלבד ה‑HTTPS endpoints המוצהרים.
- **ניהול סודות**: כל מפתחות ה‑API של צדדים שלישיים נשמרים ב‑**Google Secret Manager**, מוצפנים, ונחשפים רק בזמן ריצה לפונקציה שצריכה אותם. אין סיסמאות בקוד, ב‑Git, או בקובצי קונפיגורציה.
- **לוגים וניטור**: Cloud Logging עם שמירת לוגים מובנים וגישה מבוקרת.

## 6. היכן המידע נשמר וכמה זמן

| סוג המידע | מיקום | תקופת שמירה |
|---|---|---|
| **תוכן הודעות (היסטוריה מלאה)** | מקומי על המכשיר (SQLite) | עד שהמשתמש מוחק את האפליקציה / חשבונו |
| **תוכן הודעות בענן (מאגר זמני)** | Firestore | **≤ 48 שעות** — בשלב הבטא נמחק אוטומטית ע"י משימה מתוזמנת שרצה כל שעה |
| אנשי קשר, פריטי ידע, פעילויות | **מקומי בלבד** (SQLite) | עד מחיקה |
| הגדרות תרגום פר־איש קשר | Firestore | עד שהמשתמש מסיר אותן |
| הודעות מתוזמנות (ממתינות) | Firestore + Cloud Tasks | עד שליחה / ביטול. לאחר אישור הקליינט — נמחק מיידית |
| מצב סשן WhatsApp | Firestore (מוצפן ע"י Google Cloud) | עד ניתוק יזום |
| Push Token | Firestore | עד התנתקות / ביטול |
| חשבון משתמש (אימייל, UID, שם) | Firebase Auth | עד מחיקת חשבון |
| מדדי שימוש מצרפיים (אנונימיים) | Firestore | קבוע |

**מחיקה אוטומטית של תוכן הודעות** מתבצעת ע"י הפונקציה המתוזמנת `cleanupExpiredMessages` (שדה `expiresAt` + מחיקה ב‑batch אחת לשעה).

## 7. הצפנה — סיכום

| היכן | אופן ההצפנה |
|---|---|
| במעבר (כל קישור רשת) | TLS 1.2+ |

## 8. ספקי משנה (Subprocessors)

לכל ספק יש תפקיד תפעולי מוגדר, וחתום על המחויבויות החוזיות שלו. **אין שיתוף עם רשתות פרסום, brokers, או צדדי ג' לצרכי שיווק.**

| ספק | תפקיד | מה משותף איתו |
|---|---|---|
| **Google Firebase** (Auth, Firestore, Cloud Functions, Cloud Messaging, Cloud Tasks) | תשתית ענן | אימייל, UID, Push Token, מידע WhatsApp זמני (≤48h) |
| **Anthropic (Claude API)** | תרגום הודעות (רק כשהמשתמש הפעיל לאיש קשר ספציפי) | טקסט ההודעה בלבד — ללא מזהי משתמש, ללא טלפון, ללא מטא‑נתונים. **Anthropic אינה שומרת את הטקסט לאימון.** |
| **Apple (Sign in with Apple)** | אימות בלבד (iOS) | מזהה Apple, אימייל (או כתובת ממסר אנונימית), שם |
| **Google Identity** | אימות Google Sign‑In | אימייל, UID, שם |
| **OpsCenter / WAHA** (תפעולי שלנו) | גשר התקשורת ל‑WhatsApp | תוכן הודעות בזמן הגשר. מתנהג כ"לקוח WhatsApp נוסף" של המשתמש. |

## 9. תאימות וזכויות (GDPR / נושאי מידע)

- **אזור אירופאי**: כל המידע יושב ב‑`europe-west3` (Frankfurt, גרמניה).
- **זכויות נושא המידע**: גישה, תיקון, מחיקה, ניידות, חזרה מהסכמה — בכל עת.
- **מחיקת חשבון**: דרך האפליקציה (התנתקות + הסרה מוחקות את כל המידע המקומי) או בדרישה במייל. **המידע ב‑Backend נמחק תוך 30 יום, גיבויים מטוהרים תוך 90 יום.**
- **ביטול הסכמה לתרגום / לייבוא אנשי קשר**: בלחיצת כפתור באפליקציה.
- **לא נאספים מזהי פרסום** (IDFA / GAID).
- **לא מתבצע מעקב cross‑app / cross‑site**.
- **מדיניות פרטיות מלאה זמינה ציבורית** ומקושרת ישירות מתפריט ההגדרות באפליקציה.

## 10. מה הפתרון לא עושה

- **לא מאמן AI** על תוכן ההודעות / אנשי הקשר / הידע שלך.
- **לא מוכר נתונים** ולא משתף עם רשתות פרסום.
- **לא יוצר פרופיל התנהגותי** של המשתמש.
- **לא משתמש ב‑cookies** לצורך מעקב.
- **לא מאחסן תוכן הודעות בענן יותר מ‑48 שעות בשלב הבטא. עם המעבר לייצור מלא ההודעות ימחקו מיד לאחר קבלתן באפליקציית הלקוח**.
- **לא מאפשר למשתמש אחד לראות נתונים של משתמש אחר** — לא מכל נתיב.

## 11. שקיפות לגבי גשר WhatsApp

חשוב לנו להיות שקופים: WhatsApp עצמה היא End‑to‑End מוצפנת בין הלקוחות שלה. כשאתם מחברים את LS Duet ל‑WhatsApp, אנחנו מתפקדים כ**לקוח WhatsApp נוסף שלכם** (דרך WAHA). המשמעות:

- ההודעות עוברות בקטע אחד דרך השרת שלנו, כדי שנוכל להעביר אותן למכשיר שלכם, לתרגם (אם הפעלתם), ולתזמן.
- בקטע הזה — **הצפנה ב‑TLS, הצפנה במנוחה, ומחיקה אוטומטית תוך 48 שעות**.
- אין נתיב שבו עובד שלנו ניגש ל‑Firestore לצפייה בהודעות שלכם — Firestore Security Rules חוסמות גם את אנשי הצוות מקריאה ישירה של אוסף `whatsappMessages`. גישה תפעולית כזו הייתה דורשת שינוי כללי אבטחה מתועד ופומבי.
